    Week 4 Lesson Plan – Neural Network Basics

    1. Lesson Overview

    Learning Objectives

    By the end of this week you will be able to:

    • Explain how a single neuron (perceptron) works and why activation functions are needed.
    • Build and train a multilayer perceptron (MLP) for regression and classification using both NumPy (from scratch) and a high-level library (TensorFlow/Keras or PyTorch).
    • Describe and implement gradient descent and backpropagation conceptually and in code.
    • Use common regularization techniques (dropout, weight decay) and basic callbacks (EarlyStopping).
    • Visualize and debug training with learning curves and (optionally) TensorBoard.

    2. Core Definitions

    Term Definition Simple Example
    Neuron (Perceptron) A function that computes ( z = \mathbf{w}\cdot\mathbf{x} + b ) then passes it through an activation function. Predicting “spam vs not spam” from word counts.
    Activation Function Nonlinear function applied to neuron output (e.g., ReLU, sigmoid). ReLU keeps positives, zeroes negatives.
    Feedforward Network (MLP) Layers of neurons where outputs of one layer feed the next. Input (pixels) → Hidden → Output (digit class).
    Gradient Descent Iterative method to minimize loss by moving weights opposite the gradient. Adjust regression weights to reduce MSE.
    Backpropagation Efficient algorithm to compute gradients of all weights using the chain rule. Update all network layers after one forward pass.
    Loss Function Quantifies error between prediction and target. Cross-entropy for classification, MSE for regression.
    Regularization Techniques to prevent overfitting (dropout, L2). Randomly “drop” 20% of neurons each step.
    Epoch / Batch / Iteration Epoch: one pass over training set. Batch: subset used for one update. Iteration: one batch update. 10,000 samples, batch size 100 → 100 iterations per epoch.

    3. Concept Sections

    A. The Perceptron & Activation Functions

    Intuition (Grade‑10 friendly):
    A neuron is like a weighted “decision calculator.” Each input gets multiplied by a weight, all added up, plus a bias. If the total is big enough, the neuron “fires.”
    Without a nonlinear activation, stacking many neurons is just doing one big linear equation—boring! Activations add curves and bends so networks can learn complex patterns.

    Common Activations:

    Name Formula Range Pros Cons
    Sigmoid ( \sigma(z)=1/(1+e^{-z}) ) (0,1) Probabilities, smooth Vanishing gradients for large |z|
    Tanh ( \tanh(z) ) (−1,1) Zero-centered Still can vanish
    ReLU ( \max(0,z) ) [0,∞) Fast, simple “Dead” neurons (negative forever)
    Leaky ReLU ( \max(0.01z,z) ) (−∞,∞) Fixes dead ReLU Slightly more compute

    Why This Matters:
    - Picking the right activation affects learning speed & accuracy.
    - Output layer activation ties to loss: sigmoid + BCE for binary, softmax + cross entropy for multi-class.

    B. Gradient Descent & Backpropagation

    Story Version:
    You’re on a dark hillside (loss surface) trying to find the lowest point. You feel the slope (gradient) and take a step downhill. Repeat until the ground is mostly flat.

    Formal Idea:
    - Loss ( L(\mathbf{w}) ).
    - Update rule: ( \mathbf{w}{t+1} = \mathbf{w}_t - \eta \nabla{\mathbf{w}} L ).
    - Backprop uses chain rule to compute all partial derivatives efficiently.

    Mini Example (1 neuron, MSE):
    1. Forward: ( \hat{y} = \sigma(\mathbf{w}\cdot\mathbf{x}+b) )
    2. Loss: ( L = \frac{1}{2}(\hat{y}-y)^2 )
    3. Compute ( \partial L/\partial w_i = (\hat{y}-y)\cdot\sigma'(z)\cdot x_i )

    Why It Matters:
    Understanding the math demystifies neural nets and helps you debug exploding/vanishing gradients.

    C. Regularization & Training Tricks

    • Overfitting: Training loss ↓ but validation loss ↑.
    • Dropout: Randomly set some activations to 0 during training.
    • L2 Weight Decay: Add ( \lambda |\mathbf{w}|^2 ) to the loss to keep weights small.
    • EarlyStopping: Stop when validation doesn’t improve for N epochs.
    • Learning Rate Schedules: Reduce (\eta) over time.

    Why It Matters:
    Models generalize better, saving you from false confidence.

    5. Step-by-Step Exercises (10 total)

    Exercise 1: Single Neuron in NumPy (Binary Classification)

    Purpose: Understand forward pass & loss manually.

    import numpy as np

# Data: OR gate
X = np.array([[0,0],[0,1],[1,0],[1,1]])
y = np.array([0,1,1,1])

# Weights & bias
w = np.random.randn(2)
b = 0.0

def sigmoid(z):
    return 1/(1+np.exp(-z))

# Forward
z = X.dot(w) + b
y_hat = sigmoid(z)
loss = np.mean((y_hat - y)**2)
print("Predictions:", y_hat.round(3))
print("Loss:", loss)

    Extensions: Manually tweak w,b to see loss change.

    Exercise 2: Gradient Descent by Hand (NumPy)

    lr = 0.1
for epoch in range(5000):
    z = X.dot(w) + b
    y_hat = sigmoid(z)
    loss = np.mean((y_hat - y)**2)

    # grads
    dL_dy = 2*(y_hat - y)/len(y)
    dy_dz = y_hat*(1-y_hat)
    dz_dw = X
    dz_db = 1

    grad_w = dz_dw.T.dot(dL_dy*dy_dz)
    grad_b = np.sum(dL_dy*dy_dz*dz_db)

    w -= lr*grad_w
    b -= lr*grad_b

print("Final loss:", loss)
print("w,b:", w, b)

    Outcome: Loss decreases; weights converge.

    Exercise 3: Build a Tiny MLP in NumPy (2-2-1)

    np.random.seed(0)
W1 = np.random.randn(2,2)
b1 = np.zeros(2)
W2 = np.random.randn(2,1)
b2 = np.zeros(1)

def relu(x): return np.maximum(0,x)

for epoch in range(2000):
    # forward
    h = relu(X.dot(W1)+b1)
    y_hat = sigmoid(h.dot(W2)+b2)
    loss = np.mean((y_hat - y.reshape(-1,1))**2)

    # backprop
    dL_dy = 2*(y_hat - y.reshape(-1,1))/len(y)
    dy_dz2 = y_hat*(1-y_hat)
    dL_dz2 = dL_dy * dy_dz2
    dL_dW2 = h.T.dot(dL_dz2)
    dL_db2 = dL_dz2.sum(axis=0)

    dh = dL_dz2.dot(W2.T)
    dz1 = dh*(h>0)  # relu grad
    dL_dW1 = X.T.dot(dz1)
    dL_db1 = dz1.sum(axis=0)

    # update
    lr = 0.1
    W1 -= lr*dL_dW1
    b1 -= lr*dL_db1
    W2 -= lr*dL_dW2
    b2 -= lr*dL_db2

print("Final Loss:", loss)
print("Predictions:", (y_hat>0.5).astype(int).flatten())

    Exercise 4: MLP with Keras (TensorFlow)

    import tensorflow as tf
from tensorflow.keras import layers, models

model = models.Sequential([
    layers.Dense(8, activation='relu', input_shape=(2,)),
    layers.Dense(1, activation='sigmoid')
])

model.compile(optimizer='adam', loss='binary_crossentropy', metrics=['accuracy'])
history = model.fit(X, y, epochs=200, verbose=0)

print("Final acc:", history.history['accuracy'][-1])

# Plot learning curve (optional)
import matplotlib.pyplot as plt
plt.plot(history.history['loss'])
plt.title("Training Loss")
plt.xlabel("Epoch")
plt.ylabel("Loss")
plt.show()

    Exercise 5: Same Network in PyTorch

    import torch
import torch.nn as nn
import torch.optim as optim

X_t = torch.tensor(X, dtype=torch.float32)
y_t = torch.tensor(y.reshape(-1,1), dtype=torch.float32)

model = nn.Sequential(
    nn.Linear(2,8),
    nn.ReLU(),
    nn.Linear(8,1),
    nn.Sigmoid()
)

criterion = nn.BCELoss()
optimizer = optim.Adam(model.parameters(), lr=0.01)

for epoch in range(500):
    optimizer.zero_grad()
    y_hat = model(X_t)
    loss = criterion(y_hat, y_t)
    loss.backward()
    optimizer.step()

print("Final loss:", loss.item())
print("Pred:", (y_hat.detach().numpy()>0.5).astype(int).flatten())

    Exercise 6: Overfitting Demo (Train vs. Validation)

    Use a small noisy dataset:

    import numpy as np
np.random.seed(1)
Xn = np.linspace(-1,1,200).reshape(-1,1)
y_true = Xn**3
y_noise = y_true + 0.2*np.random.randn(*y_true.shape)

# Train/val split
X_train, X_val = Xn[:150], Xn[150:]
y_train, y_val = y_noise[:150], y_noise[150:]

import tensorflow as tf
model = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activation='relu', input_shape=(1,)),
    tf.keras.layers.Dense(64, activation='relu'),
    tf.keras.layers.Dense(1)
])
model.compile(optimizer='adam', loss='mse')
hist = model.fit(X_train, y_train, validation_data=(X_val, y_val),
                 epochs=200, verbose=0)

import matplotlib.pyplot as plt
plt.plot(hist.history['loss'], label='train')
plt.plot(hist.history['val_loss'], label='val')
plt.legend(); plt.title("Overfitting Demo")
plt.show()

    Exercise 7: Dropout & Weight Decay

    Compare models with and without regularization.

    from tensorflow.keras import regularizers

model_reg = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activation='relu', input_shape=(1,),
                          kernel_regularizer=regularizers.l2(1e-3)),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(64, activation='relu',
                          kernel_regularizer=regularizers.l2(1e-3)),

])

model_reg.compile(optimizer='adam', loss='mse')
hist2 = model_reg.fit(X_train, y_train, validation_data=(X_val, y_val),
                      epochs=200, verbose=0)

    Exercise 10: Mini Project – MNIST Digit Classifier (Lite Version)

    1. Load MNIST (digits 0–9).
    2. Normalize pixels to [0,1].
    3. Build a 2‑hidden‑layer MLP.
    4. Train, plot accuracy.
    5. Show 10 random predictions.
    import tensorflow as tf
(X_train, y_train), (X_test, y_test) = tf.keras.datasets.mnist.load_data()
X_train = X_train.reshape(-1,784)/255.0
X_test  = X_test.reshape(-1,784)/255.0

model = tf.keras.Sequential([
    tf.keras.layers.Dense(256, activation='relu', input_shape=(784,)),
    tf.keras.layers.Dropout(0.2),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dense(10, activation='softmax')
])
model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
hist = model.fit(X_train, y_train, epochs=5, batch_size=128, validation_split=0.1, verbose=1)
print("Test acc:", model.evaluate(X_test, y_test, verbose=0)[1])
