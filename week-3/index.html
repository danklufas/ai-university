    Week 3 Lesson Plan – Unsupervised Learning: Clustering & Dimensionality Reduction

    1. Lesson Overview

    Learning Objectives
    By the end of this week you will be able to:

    • Explain the difference between supervised and unsupervised learning and when to use each.
    • Describe and implement k‑means and hierarchical clustering, and evaluate clusters (elbow method, silhouette score).
    • Perform Principal Component Analysis (PCA) to reduce dimensionality and visualize high‑dimensional data.
    • Use scikit‑learn to build unsupervised pipelines and interpret results for business use cases (e.g., customer segmentation).
    • Communicate findings with appropriate plots (scatter matrices, dendrograms, scree plots).

    2. Core Definitions

    Term Definition Simple Example
    Unsupervised Learning ML that finds structure/patterns in unlabeled data. Grouping customers by behavior without knowing “true” segments.
    Clustering Grouping similar data points together. Clustering products by sales patterns.
    k‑means Algorithm that partitions data into k clusters by minimizing within‑cluster variance. Choosing 3 customer segments by spending habits.
    Hierarchical Clustering Builds a tree of clusters (merging or splitting). Dendrogram showing how customers merge into bigger groups.
    PCA (Principal Component Analysis) Linear technique that projects data into fewer dimensions while preserving as much variance as possible. Reducing 30 questionnaire answers to 3 principal factors.
    Silhouette Score Measure of how well a point fits in its cluster vs. others (−1 to 1). Average silhouette ~0.6 suggests good separation.
    Elbow Method Heuristic plot of inertia vs. k to pick an optimal number of clusters. “Elbow” at k=4 means adding more clusters gives diminishing returns.

    3. Concept Sections

    A. k‑Means Clustering (Deep Dive)

    Intuition (Grade‑10 friendly):
    Imagine you have a bunch of dots on paper. You pick k “centers” and pull nearby dots toward each center. Then you move each center to the middle of its dots. Repeat until things stop moving much.

    Formal Steps: 1. Choose k (number of clusters).
    2. Initialize k centroids (randomly or with k‑means++).
    3. Assignment step: Assign each point to the closest centroid (Euclidean distance).
    4. Update step: Recompute each centroid as the mean of assigned points.
    5. Repeat 3–4 until assignments stop changing or max iterations hit.

    Cost Function (Inertia):
    Minimize the sum of squared distances between each point and its centroid.

    Why It Matters:
    Fast, simple, works well for spherical clusters. Great first pass for segmentation.

    Limitations:
    - You must pick k (not always obvious).
    - Sensitive to outliers and scaling.
    - Assumes roughly equal-sized, spherical clusters.

    B. Hierarchical Clustering

    Intuition:
    Start with each point as its own cluster. Gradually merge the two closest clusters until only one remains. The history of merges is a dendrogram (tree diagram).

    Two main flavors: - Agglomerative (bottom‑up): Start with singletons, merge upwards.
    - Divisive (top‑down): Start with one big cluster, split apart.

    Linkage Criteria:
    Defines “distance” between clusters:
    - Single linkage: closest pair of points.
    - Complete linkage: farthest pair.
    - Average linkage: average distance between all pairs.
    - Ward’s method: minimizes increase in total within-cluster variance.

    Why It Matters:
    - No need to pre-specify k. You can “cut” the dendrogram later.
    - Reveals nested structure.

    Limitations:
    - Slower than k‑means on very large datasets.
    - Choice of linkage can change results noticeably.

    C. Principal Component Analysis (PCA)

    Intuition (simple):
    If you have lots of columns (features), many may overlap. PCA finds new axes (principal components) that capture the most “spread” (variance). You can keep the first few and ignore the rest.

    Step-by-Step (conceptual): 1. Standardize features (mean 0, std 1).
    2. Compute covariance matrix of data.
    3. Find eigenvalues & eigenvectors (directions of maximum variance).
    4. Sort eigenvectors by eigenvalues (largest first).
    5. Project data onto the top d eigenvectors to reduce dimensions.

    Key Plots: - Scree plot: eigenvalue (variance explained) vs. component index.
    - Explained variance ratio: how much variance each component captures.

    Why It Matters:
    - Visualization: reduce to 2D/3D for scatter plots.
    - Noise reduction & faster modeling.
    - Feature engineering (components as new features).

    Limitations:
    - Linear; may not capture complex nonlinear relationships.
    - Components are combinations of original features—sometimes hard to interpret.

    5. Step-by-Step Exercises (10 total)

    Follow the same pattern as Week 1 exercises: overview/purpose → concept reinforcement → real-world relevance → step-by-step code → expected outcomes → extensions → notes.

    Exercise 1: k‑Means on a Toy 2D Dataset (Visual)

    Overview & Purpose
    Run k‑means on simple 2D points and plot clusters to see the algorithm visually.

    Concept Reinforcement
    - k‑means assignment/update loop
    - Euclidean distance, centroids
    - Need for feature scaling

    Real World Relevance
    Customer segmentation by 2 numeric features (e.g., spend & visits).

    Step-by-Step Instructions

    import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

# 1. Create toy data
np.random.seed(42)
X1 = np.random.normal([2, 2], 0.3, size=(50, 2))
X2 = np.random.normal([6, 6], 0.4, size=(50, 2))
X3 = np.random.normal([2, 7], 0.5, size=(50, 2))
X = np.vstack([X1, X2, X3])

# 2. Scale (optional but recommended)
scaler = StandardScaler()
Xs = scaler.fit_transform(X)

# 3. Fit KMeans
kmeans = KMeans(n_clusters=3, random_state=42)
labels = kmeans.fit_predict(Xs)

# 4. Plot
plt.scatter(Xs[:,0], Xs[:,1], c=labels, cmap='viridis', s=30)
plt.scatter(kmeans.cluster_centers_[:,0], kmeans.cluster_centers_[:,1],
            c='red', marker='x', s=100, label='centroids')
plt.legend(); plt.title("k-means Clusters (scaled data)")
plt.show()

    Expected Outcomes & Interpretation
    Three groups colored differently, centroids marked. Check if they match where you generated data.

    Extensions
    - Try different seeds or cluster counts.
    - Remove scaling and compare results.

    Notes
    random_state for reproducibility. Scaling often critical.

    Exercise 2: Elbow Method to Choose k

    Purpose: Plot inertia vs. k.

    inertias = []
for k in range(1, 10):
    km = KMeans(n_clusters=k, random_state=42).fit(Xs)
    inertias.append(km.inertia_)

plt.plot(range(1,10), inertias, marker='o')
plt.xlabel("k"); plt.ylabel("Inertia (Sum of Squared Distances)")
plt.title("Elbow Method")
plt.show()

    Expected Outcome
    Look for the “elbow” (sharp bend). That k is a good starting point.

    Exercise 3: Silhouette Score Evaluation

    Purpose: Quantify cluster quality.

    from sklearn.metrics import silhouette_score

for k in range(2, 7):
    km = KMeans(n_clusters=k, random_state=42).fit(Xs)
    score = silhouette_score(Xs, km.labels_)
    print(f"k={k}, silhouette={score:.3f}")

    Outcome: Higher average silhouette indicates better separation.

    Exercise 4: Hierarchical Clustering & Dendrogram

    Purpose: Visualize merges with scipy.

    from scipy.cluster.hierarchy import linkage, dendrogram
import matplotlib.pyplot as plt

Z = linkage(Xs, method='ward')  # or 'single', 'complete', 'average'
plt.figure(figsize=(8,4))
dendrogram(Z, truncate_mode='lastp', p=12, leaf_rotation=45., leaf_font_size=10.)
plt.title("Hierarchical Clustering Dendrogram (truncated)")
plt.xlabel("Cluster size"); plt.ylabel("Distance")
plt.show()

    Outcome: Tree plot; cut at a distance to choose clusters.

    Exercise 5: AgglomerativeClustering in scikit‑learn

    Purpose: Fit hierarchical clustering and compare to k-means.

    from sklearn.cluster import AgglomerativeClustering

agg = AgglomerativeClustering(n_clusters=3, linkage='ward')
labels_agg = agg.fit_predict(Xs)

plt.scatter(Xs[:,0], Xs[:,1], c=labels_agg, cmap='viridis', s=30)
plt.title("Agglomerative Clustering (Ward)")
plt.show()

    Outcome: Similar but not identical groupings vs. k-means.

    Exercise 6: PCA on a 4‑Feature Dataset

    Purpose: Reduce 4D to 2D and visualize.

    from sklearn.decomposition import PCA

# Make a small 4D toy set
np.random.seed(0)
X4 = np.random.randn(200, 4) @ np.array([[1, 0.2, 0.1, 0],
                                         [0.2, 1, 0.3, 0.1],
                                         [0.1, 0.3, 1, 0.2],
                                         [0, 0.1, 0.2, 1]])
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X4)

print("Explained variance ratio:", pca.explained_variance_ratio_)

plt.scatter(X_pca[:,0], X_pca[:,1], s=20)
plt.title("PCA Projection (2D)")
plt.xlabel("PC1"); plt.ylabel("PC2")
plt.show()

    Outcome: 2D scatter and variance ratios.

    Exercise 7: Scree Plot (Explained Variance)

    pca_full = PCA().fit(X4)
plt.plot(np.arange(1, X4.shape[1]+1), np.cumsum(pca_full.explained_variance_ratio_), marker='o')
plt.xlabel("Number of Components")
plt.ylabel("Cumulative Explained Variance")
plt.title("Scree / Cumulative Variance Plot")
plt.grid(True)
plt.show()

    Outcome: Choose number of components capturing, say, 90% variance.

    Exercise 8: Reconstruct Data from First PCs

    Purpose: Show information loss.

    k = 2  # keep first 2 components
pca_k = PCA(n_components=k).fit(X4)
X_red = pca_k.transform(X4)
X_recon = pca_k.inverse_transform(X_red)

mse = np.mean((X4 - X_recon)**2)
print("Reconstruction MSE:", mse)

    Outcome: Lower MSE if more components kept.

    Exercise 9: Combine PCA + k‑Means Pipeline

    from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ("scale", StandardScaler()),
    ("pca", PCA(n_components=2)),
    ("kmeans", KMeans(n_clusters=3, random_state=42))
])
pipe.fit(X4)
print("Clusters:", np.unique(pipe['kmeans'].labels_))

    Outcome: Clean pipeline workflow (preprocessing + model).

    Exercise 10: Business Mini‑Project – Customer Segmentation

    Purpose: Apply to a CSV of customer metrics (spend, visits, tenure, channel usage).

    1. Load & clean data.
    2. Scale numeric features.
    3. Try k from 2–8, elbow + silhouette.
    4. Pick best k, interpret clusters (means of features).
    5. Visualize with PCA 2D scatter colored by cluster.
    6. Write 3–4 bullet “personas” describing each segment.

    Skeleton:

    import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from sklearn.decomposition import PCA

df = pd.read_csv("customers.csv")  # replace with your file
features = ["annual_spend", "visits", "tenure_months", "returns"]
X = df[features].dropna()

X_scaled = StandardScaler().fit_transform(X)

scores = {}
for k in range(2,9):
    km = KMeans(n_clusters=k, random_state=42).fit(X_scaled)
    scores[k] = silhouette_score(X_scaled, km.labels_)
print(scores)

best_k = max(scores, key=scores.get)
km = KMeans(n_clusters=best_k, random_state=42).fit(X_scaled)
df["segment"] = km.labels_

# PCA for visualization
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

plt.scatter(X_pca[:,0], X_pca[:,1], c=km.labels_, cmap='tab10', s=20)
plt.title(f"PCA projection colored by k-means (k={best_k})")
plt.show()

print(df.groupby("segment")[features].mean())

    Outcome: Table of segment means + plots.

    Extensions
    - Try hierarchical clustering on same data.
    - Use different features or include categorical encodings.
